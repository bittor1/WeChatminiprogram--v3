🔧 Detail页面头像显示修复

解决了数据字段名不匹配导致的头像、排名、姓名不显示问题

🎯 问题根源

❌ 数据字段名不匹配

问题描述：

  • WXML模板使用 entryDetail.avatar、entryDetail.rank、entryDetail.name
  • 但JS代码将数据存储在 userInfo 字段中
  • 导致头像、排名、姓名都无法显示

🔄 修复对比

❌ 修复前（错误）

<image src="{{entryDetail.avatar}}" /> <view>#{{entryDetail.rank}}</view> <text>{{entryDetail.name}}</text>

问题：引用不存在的 entryDetail 字段

✅ 修复后（正确）

<image src="{{userInfo.avatar}}" /> <view>#{{userInfo.rank}}</view> <text>{{userInfo.name}}</text>

修复：使用正确的 userInfo 字段

📋 修复详情

修改的文件：

pages/detail/detail.wxml

具体修改：

  • ✅ {{entryDetail.avatar}} → {{userInfo.avatar}}
  • ✅ {{entryDetail.rank}} → {{userInfo.rank}}
  • ✅ {{entryDetail.name}} → {{userInfo.name}}

数据来源确认：

// detail.js 中的 loadUserDetail 函数 this.setData({ userInfo: user // ← 数据存储在 userInfo 中 });
